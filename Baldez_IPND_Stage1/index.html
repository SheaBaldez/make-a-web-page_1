The Basics

What is the Internet?

Basically, the Internet is a collection of documents which can be linked to eachother

Learning to Code

Learning to code is like learning another language, only much easier. Unlike learning a human language, there are no irregular verbs, no cultural connotations or false cognates and you do not have to practice a new accent. You also don't have to memorize much because you can always look up whatever phrase you're trying to use. All you have to do is understand the basic, overarching rules of talking to computers and you have to get an understanding of how the computer "understands" what you say to it. Non-programmers see familiar things like images, text and videos on a webpage but, a programmer knows all of those images, text and videos are appearing only because someone gave the computer specific and structured instructions to display those things. Since they can speak the same language as a computer they can see how the webpage might look from the computer's perspective. This is known as
technological empathy
.

What is the Computer's Perspective?

Computers are literal. They don't jump to conclusions about what you're trying to say to them. They do exactly as they are told; so it doesn't matter what the programmer meant, it only matters what the programmer said. For instance, if I told you to "take the stairs" you would fill in the blanks automatically. You would know I meant for you to find the stairway, choose a direction to go, then step on the first step, then the next and so on until you reached your destination. You would be able to do this because we rely on context and prior knowledge in order to understand each other. A computer is not so sympathetic to you. It doesn't know what you want and it won't make any assumptions so you have to be very specific with your instructions. If a computer could "take the stairs" you would have to tell it exactly how to complete each stage in the process.

HTML

What is HTML?

HyperText Markup Language
is the language of the Internet. It is made up of text, documents and markup (the commands dictating the appearance of the text, image, etc.)

Page Structure

Pages are organized in a way that, when visualized, might look like a chart which starts with one box and branches out into many boxes in a systematic way. If we were to draw it out, it may look like tree roots growing outward and multiplying. This is called the
Document Object Model
in the programming world. The Wikipedia page on Tree Structure has a lot more information on tree structures but, for me, the most helpful graphic Wikipedia provides is this one:
The image shows the way an encyclopedia visually represented all of its topics and subtopics. This is similar in structure to a document for a webpage in which you have one main section then smaller sections fit into it.

The Anatomy of an HTML Document

An HTML document is structured like this:
  • <!DOCTYPE HTML>
    This tells us what kind of document it will be
  • <html>
    This indicates the beginning of the HTML
  • <head> </head>
    This section can contain the title
    <title></title>
    of the page as well as the links to the CSS, Javascript, or metadata for the page.
  • <body>
    </body>
    This section contains the contents of the page.
  • </html>
    This indicates the end of the HTML content

Some HTML Terminology

  • Void tag:
    a tag which does not contain content (such as the line break tag
    <br>
    ) so it does not require a closing tag.
  • Inline tag:
    tags which are just text; they don't change the size or position of the content; for example
    <b>
    ,
    <em>
    ,
    <img>
  • Block tag:
    a tag which creates boundries or an invisible box; it can have a height and width;
    <p>
    is a block tag
  • Element:
    the group of text and tags which includes the opening and closing tags and the content

Avoiding Repetition

Have you ever had to nag someone to do something or remind someone over and over of how they're supposed to get something done? It's maddening. The same goes for talking to computers. When you tell a computer to do something, you must be specific and detailed with your words. If you want the computer to do the same thing in multiple parts of a webpage, you will have to type all of this out each time. This can get very tiresome and create a lot of unnecessary code in a page or website with even small amounts of content. When we have to repeat ourselves over and over, we also risk making mistakes or skipping it in places we did not mean to leave it out. It's better if you can create a shortcut by giving the computer a set of attributes it should apply whenever you use the shortcut tag. These shortcut tags are normally either a
<div>
or a
<span>
.

CSS

What is CSS?

Cascading Style Sheets
create the look and feel of the webpage. If a webpage were a tent, the HTML would be the wooden frame and the CSS would be the fabric that covers it. CSS is also a way of avoiding repetition. For instance, if you want every heading to have the same font, color, size and background color, you do not have to write this into the code of each heading repeatedly. Instead, you create a CSS file with the style information and link that file to your HTML file. Here are the basic steps:
  1. 1. Create two separate files in one folder on your computer. One is an HTML (.html) document and the other is a CSS (.css) document.
  2. 2. Write the HTML which includes the structure and content of the page. Make sure to identify certain
    elements
    to which you would like to apply
    attributes
    from your CSS document.
  3. 3. Write the CSS which includes information on the colors, fonts, visual layout, etc.
  4. 4. Put a link to the CSS file into the
    <head>
    section of your HTML document that looks like this:
    <link href="thenameofyourcssfile.css" rel="stylesheet" type="text/css">
Now your CSS document can talk to your HTML document!
